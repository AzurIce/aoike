# Aoike

啃了点 [MkDocs](https://github.com/mkdocs/mkdocs) 的码然后搓出的一个静态博客生成器。

核心理念就是利用目录组织结构来减少配置，
比如让目录结构对应分类（替代 YAML 头的 `categories`），
让以 `_` 开头的文件/目录被忽略（替代 YAML 头的 `draft`）。

目前还有一些内容需要借助 YAML 头配置，
不过目标就是将这部分配置最小化。

「写博客时不该操心文章之外的各种繁琐的事情」

## 使用

### 1. 安装

目前还没有上传到 PyPI（因为目前实在是简陋），
想要试试的话就先 `clone` 到本地，然后在本地 `pip install <Aoike目录>`，
当然如果你想要 Hack 一下的话就 `pip install -e <Aoike目录>`。

### 2. 使用

将你的博客目录结构组织为如下的样子：

```
<博客目录>
└─posts
   ├─<category1>
   │  ├─<subcategory1>
   │  │  └─<post1>
   │  ├─<subcategory2>
   │  │  └─<post2>
   │  ├─<post3>
   │  └─<post4>
   ├─<post5>
   └─<post6>
```

也就是将所有的 `.md` 源文件放在 `posts/` 目录内，如果想要对其进行分类就创建对应的目录结构。

以 `_` 开头的目录、文件会被忽略。

然后在博客目录下执行：

```shell
aoike build
```

之后构建的网站会被放在 `site/` 目录下（自动创建）。

所有文件的相对位置都不会改变。

所有的 `.md` 文件会被使用模板渲染后生成在对应位置，其他类型的文件会被直接复制过去。
